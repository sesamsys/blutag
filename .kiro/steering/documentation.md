# Documentation Standards

## Documentation Location
All project documentation should be stored in the `/docs` directory at the root of the project.

## Documentation Types

### Project Documentation (`/docs`)
- Architecture decisions and design documents
- Content structure and organization guides
- Improvement proposals and technical debt tracking
- API documentation
- Deployment guides
- Any other project-level documentation

### Code Documentation
- Inline comments for complex logic
- JSDoc/TSDoc for functions and components when needed
- README.md remains at root for quick project overview

## Documentation Practices

### When to Document
- Document architectural decisions when they're made
- Track improvements and technical debt as they're identified
- Update documentation when making significant changes
- Create guides for non-obvious workflows

### Documentation Format
- Use Markdown (.md) for all documentation
- Use clear, descriptive filenames (kebab-case)
- Include a table of contents for longer documents
- Keep documentation close to the code it describes when possible

### Maintenance
- Review and update documentation during feature development
- Remove outdated documentation promptly
- Link related documentation together
- Keep a changelog for significant project changes
